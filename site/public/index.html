CodeCrafters
The project team consists of six second year ICT students, Henni Hiltunen, Phat Vuong, Petri Peltomaa, Eelis Suhonen, Ijini Lekamge and Nimesha Fernando. Each member of the team has a specific role within the project. All of the team members have some experience in the field, mostly from studies, therefore the team is taking this project as a chance to broaden their knowledge, get more experience and learn more useful skills for the working life.
The assignment is to enhance and add new features for the Tukko 1.0 traffic visualizer. Tukko 1.0 was made in the summer 2023 WimmaLab by the IoTitude team. Tukko 1.0 is and open-source service that utilizes public traffic APIs, specifically the Digitraffic API, to gather real-time traffic data. Tukko 1.0 unifies the scattered info across multiple endpoints into it's own simpler API. This data is then shown in a React based user interface, with mapping tools by Leaflet utilizing multiple plugins such as Geoman and Cluster markers. Users can select vehicle types and various timescales. Utilizing custom icons, popups and tooltips, Tukko strives to convey information intuitively. Tukko 1.0 was implemended with the following technologies: React, TypeScript, NodeJS, Docker, Kubernetes, CI/CD GitLab, Vite, REST, Leaflet.js, and Digitraffic APIs.

SOLUTION

Tukko Traffic Visualizer
Tukko 1.1 is an enhanced and improved version of it's preceder Tukko 1.0. It is more user friendly and accessible, and the security measures are hardened. The products end user demograph consists of any type of users seeking real-time traffic information, for example transport coordinators, delivery drivers and commuters.
